# タスク: Discord通知の重複送信修正

## 概要
脚本のアップロード/更新時に、Discordへの通知が重複して（3件など）送信される問題を修正します。

## 課題
- 現状、Discord通知のリトライロジックにおいてタイムアウト（デフォルト5秒）が発生した場合、POSTリクエストであっても再試行されるようになっている。
- PDF生成やファイルアップロードを含む通知は、5秒を超過する可能性が高く、その結果、Discord側では処理が成功していてもクライアント側でタイムアウトしてしまい、重複して送信されてしまう。

## 作業内容
- [x] 調査と原因特定
- [ ] `backend/src/services/discord.py` の `DiscordService` における `httpx.AsyncClient` のタイムアウト設定を延長する（5秒 -> 30秒）。
- [ ] リトライ回数の調整またはより厳密なエラー判定の検討。
- [ ] 修正内容の動作確認。

## 完了条件
- 脚本更新時にDiscord通知が1回のみ送信されること。
