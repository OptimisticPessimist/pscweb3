# 実装・検証ウォークスルー (香盤表のキャラクターソート)

## 実装内容
この機能では、香盤表で表示されるキャラクターの並び順を管理するために、以下の変更を行いました。

1. **DBモデル (`Character`) の拡張**:
   - `models.py` に `order` (Integer) を追加。
   - `alembic` 経由で `server_default="0"` を伴うマイグレーションファイルを作成し適用。

2. **スクリプト・パーサーの改修** (`fountain_parser.py`):
   - スクリプトを読み込む際に、まず「`# 登場人物`」セクションを抽出。そこで列挙されている順序に若い数値を `order` として割り振り。
   - その後、スクリプト本文を解析し、セリフや行動で新たなキャラクターが登場した際には、それに続く `order` を割り当てる仕様に変更しました。

3. **APIレスポンスの統合**:
   - `schemas/scene_chart.py` の `CharacterInScene` に `order` プロパティを追加。
   - `SceneChartResponse` を組み立てる際、該当する `mapping.character.order` から値を取得してフロントに送るよう結合。

4. **フロントエンド描画の最適化** (`SceneChartPage.tsx`):
   - 香盤表マトリックス生成時、全シーンからユニークなキャラクター集合 `uniqueChars` を構築する際、各自の `order` フィールドを保持。
   - 描画直前に `.sort((a, b) => a.order - b.order)` を実行し、見出し・データ列ともに常に同じ順序（登場人物欄＞登場順）になるよう実装しました。

## テスト内容
- **ユニットテスト**: 提供されている `test_fountain_parser.py` を実行。既存のテストでエラーが出ないこと、かつ新しい機能が反映されることを確認。
  - `uv run pytest tests/unit/test_fountain_parser.py -v` で全て Passed。

- **マニュアル検証設計**:
  以下のような順番のFountainファイルを解析し、`SceneChart`取得時にキャラクターが「探偵→被害者→容疑者A→容疑者B→容疑者C」としてソートされることを意図した変更が反映されています。
  ```fountain
  Title: テスト台本

  # 登場人物
  探偵
  被害者
  容疑者A
  容疑者B
  容疑者C

  # 第1幕
  ## 第1場

  @探偵
  1
  
  @容疑者A
  3
  ```

## デモンストレーション
- **before** : 登場した順番や、Setオブジェクト等に基づく不安定な（あるいは想定外の初使用順）ソートで列が定義。
- **after**  : 設定した「登場人物」の定義がしっかり表示優先順位として処理され、綺麗に整列した香盤表テーブルを描画。
