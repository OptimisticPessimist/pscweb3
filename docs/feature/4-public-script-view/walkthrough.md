# 公開脚本閲覧・インポート機能 実装完了

## 変更内容

### Backend
- **新しいAPIエンドポイントの追加**:
    - `GET /api/public/scripts/{script_id}`: 認証なしでアクセス可能な脚本詳細取得API。`is_public=True`の脚本のみ返します。
    - `POST /api/projects/import-script/{script_id}`: 公開脚本を元に新しいプロジェクトを作成するAPI。プロジェクト作成上限（2つ）のチェックを含みます。
- **ルーター登録**: `src/main.py` に `src.api.public` を追加。

### Frontend
- **新しいページ**: `src/features/scripts/pages/PublicScriptPage.tsx`
    - 公開脚本を閲覧するための専用ページ。
    - PDFダウンロードボタンや編集機能を除外し、閲覧に特化。
    - 「この脚本を使う（Import）」ボタンを配置。
- **ルーティング**: `/public/scripts/:scriptId` を追加。
- **APIクライアント**: `scriptsApi` に `getPublicScript` と `importScript` を追加。
- **型定義**: `ScriptSummary` に `author` フィールドを追加。
- **翻訳**: 日英の翻訳ファイルに新機能用のテキストを追加。

## 検証項目

### 1. 公開脚本の閲覧
- [ ] ログアウトした状態で `/public/scripts/{script_id}` にアクセスできること。
- [ ] 脚本の内容（タイトル、著者、キャラクター、シーン）が正しく表示されること。
- [ ] PDFダウンロードボタンが表示されていないこと。

### 2. 非公開・存在しない脚本へのアクセス
- [ ] `is_public=False` の脚本IDにアクセスした場合、エラー（403/404）が表示されること。
- [ ] 存在しないIDにアクセスした場合、404エラーが表示されること。

### 3. 脚本のインポート
- [ ] ログイン状態で公開ページにアクセスし、「この脚本を使う」ボタンをクリックできること。
- [ ] インポート確認ダイアログが表示されること。
- [ ] インポート成功後、新しいプロジェクトの脚本ページにリダイレクトされること。
- [ ] プロジェクト一覧に新しいプロジェクト（"Original Title (Copy)"）が追加されていること。

### 4. プロジェクト作成上限
- [ ] 既に2つのプロジェクトを持っているユーザーがインポートしようとした場合、エラーメッセージが表示されること。
