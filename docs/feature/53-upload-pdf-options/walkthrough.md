# 修正内容の確認 (Walkthrough.md) - タスク

## 変更の概要
脚本アップロード時にもPDFの設定（用紙の向き：横置き/縦置き、文字方向：縦書き/横書き）を選択できるようにし、その設定をシステム内に保存してPDFダウンロード時の初期値として用いるように拡張しました。

## 主な変更点

### 1. データベース: `Script`テーブルへの設定保存
- `Script`モデル(`backend/src/db/models.py`)に以下の2カラムを追加しました：
  - `pdf_orientation` (デフォルト: `landscape`)
  - `pdf_writing_direction` (デフォルト: `vertical`)
- Alembicマイグレーションスクリプトを作成・実行し、DBスキーマを更新しました。

### 2. バックエンド: 設定の保存と引数の受け渡し
- スクリプト保存ロジック(`backend/src/services/script_processor.py`)の`get_or_create_script`や`process_script_upload`を更新し、引数で渡されたPDFの設定をDBへ保存するようにしました。
- `upload_script` APIエンドポイント(`backend/src/api/scripts.py`)にPDF設定のフォームパラメータを受け取る処理を追加しました。
- `download_script_pdf`API において、クエリパラメータが送られて来なかった場合に、DBに保存されたPDF設定をデフォルト値として使用するように修正しました。

### 3. フロントエンド: アップロード画面とダウンロードUIの連携
- アップロードコンポーネント(`ScriptUploadPage.tsx`)にPDFのレイアウトを選択できるプルダウンを追加し、フォーム送信時にその設定値をAPIへ引き渡す処理を追加しました。
- また、すでにアップロード済みの脚本で再アップロードする場合には、前回のPDFレイアウト設定が初期選択されるように連携しています。
- ダウンロードボタンのモーダル(`ScriptListPage.tsx`, `ScriptDetailPage.tsx`)を開いた際に、保存済みの設定がプルダウンの初期値となるように挙動を修正しました。
- `ScriptSummary`インターフェースを含め、複数のPydantic/TypeScript側の型定義を拡張しました。
