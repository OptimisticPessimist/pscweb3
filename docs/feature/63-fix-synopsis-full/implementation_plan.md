# 台本ビューアーでのあらすじ表示修正

あらすじ（シーン番号0）において、内容が全文表示されない、あるいは特定の要素（移行指示、セリフ、括弧書きなど）が欠落する問題を修正します。

## 現状の課題
1. **パーサーの挙動**: `fountain-python` ライブラリの仕様により、要素間に空行がない場合、特定の要素（あらすじ要素 `=` の直後の行など）がパースされずにスキップされることがある。
2. **フロントエンドの表示**: 台本詳細画面および公開画面の「あらすじ」セクションにおいて、`Scene.description` のみを表示しており、`Line` モデルとして保存された他の要素（移行指示、セリフ等）が表示されていない。
3. **収集ロジックの制限**: バックエンドで `Scene.description` を収集する際、`Action` と `Synopsis` 以外の要素（`Transition` 等）を `description` に含めていない。

## 提案する変更

### バックエンド: `backend/src/services/fountain_parser.py`

#### 1. プリプロセスの改善
- Fountainパーサーに渡す前に、あらすじ要素 (`=`) や移行指示 (`>`) などの後に空行を挿入するようにプリプロセスを強化し、パースの失敗を防ぎます。
- キャラクター行 (`@`) の前にも空行を確保するようにし、ブロックの区切りを明確にします。

#### 2. `description` 収集ロジックの改善
- シーン番号 0（あらすじ）の場合、全ての要素タイプ（セリフ、括弧書き、移行指示等）を `Scene.description` に含めるように修正します。これにより、既存のフロントエンドの表示ロジックでも概ね全文が表示されるようになります。

### フロントエンド

#### 1. `frontend/src/features/scripts/pages/ScriptDetailPage.tsx`
- あらすじセクションにおいて、`description` だけでなく `lines` も考慮して表示するように修正するか、`description` が不十分な場合に備えて統合的な表示を行います。
- 基本的にはバックエンドの `description` 収集を強化することで対応可能ですが、予備的に `lines` の存在を確認するようにします。

#### 2. `frontend/src/features/scripts/pages/PublicScriptPage.tsx`
- 詳細画面と同様の修正を行い、公開画面でもあらすじが全文表示されるようにします。

## 検証計画

### 1. 自動テスト
- `backend/repro_synopsis.py` を再度実行し、空行がない場合でも全ての行が `Scene.description` に含まれ、かつ `Line` として保存されることを確認します。
- `backend/tests/unit/test_fountain_parser.py` 等の既存テストを通します。

### 2. 手動検証
- 実際に空行のないあらすじを含むFountainファイルをアップロードし、台本ビューアーおよび公開画面で全文が表示されることを確認します。
- 特定の記法（移行指示 `>` やセリフ `@CHARACTER Dialogue`）があらすじ内に含まれる場合に正しく表示されるか確認します。
