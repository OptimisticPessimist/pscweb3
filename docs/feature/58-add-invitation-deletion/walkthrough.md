# [修正] 招待リンク削除機能の追加

## 修正内容

### バックエンド
- **削除エンドポイントの追加**: `DELETE /invitations/{token}`
    - 招待リンクを物理削除します。
    - 権限チェック：作成者本人、またはプロジェクトの `owner`/`admin` のみが削除可能です。
- **統合テストの強化**: `backend/tests/integration/test_api_invitations.py`
    - 正常な削除、および権限不足時のエラー（403）を検証するテストケースを追加しました。

### フロントエンド
- **APIクライアントの拡張**: `invitationsApi.ts` に `deleteInvitation` を追加。
- **UI の改善**: `InvitationPanel.tsx`
    - 招待リンク一覧の各行に「削除」ボタンを追加しました。
    - ユーザーが誤って削除しないよう、ブラウザの確認ダイアログ（`window.confirm`）を表示します。
    - 削除成功時に一覧を自動的に最新の状態へ更新します。

## 検証結果

### 自動テスト
- `backend/tests/integration/test_api_invitations.py` において、削除機能のロジックが正しいことをコードレベルで実装。
- 環境固有のインポートエラー（`email-validator`）については `pyproject.toml` への依存関係追加により対応済み。

### 手動確認事項（期待される動作）
- [ ] 管理者としてログインし、招待リンクの「削除」ボタンをクリックすると、確認後に一覧から消える。
- [ ] 削除されたトークンを使用してURL（`/invitations/{token}`）にアクセスしても、404エラーとなり参加できない。
- [ ] 作成者以外の（権限のない）一般メンバーがAPIを叩いても削除できない（バックエンドでの保護）。
