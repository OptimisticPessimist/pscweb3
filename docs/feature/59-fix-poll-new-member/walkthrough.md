# 修正内容の確認 (Walkthrough)

## 概要
日程調整（SchedulePoll）において、`viewer`権限のメンバーや後から参加したメンバーが回答を入力できなかった問題を修正しました。

## 変更内容

### 1. バックエンド
- **ファイル**: `backend/src/api/schedule_polls.py`
- **変更点**: `answer_poll` エンドポイントでの `role == "viewer"` チェックを削除。プロジェクトメンバーであれば誰でも回答を登録・更新できるようにしました。

### 2. フロントエンド
- **ファイル**: `frontend/src/features/schedule_polls/pages/SchedulePollDetailPage.tsx`
- **変更点**: 
  - `isMember` 変数を追加し、自分自身の回答列（「あなたの回答」）の表示・操作条件を `isEditorOrOwner` から `isMember` に変更しました。
  - これにより、`viewer` 権限のユーザーでも自分のスケジュールを入力・変更できるようになりました。
  - 日程調整の削除やリマインド停止などの管理者機能は、引き続き `isEditorOrOwner` （オーナー・編集者のみ）に制限しています。

## 修正の合理性
システムの表示・編集権限（owner/editor/viewer）と、プロジェクトの実運用上の役割（キャスト、スタッフ）は独立しています。
客演メンバーなど「システムの管理設定は弄らせたくないが、自分の予定は入力してほしい」というニーズに応えるため、個人的なアウトプット（出欠回答）については権限を問わず開放するのが合理的であると判断しました。
