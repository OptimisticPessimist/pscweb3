# 実装計画: CI不具合（Ruff）の最終修正

## 概要
CI (GitHub Actions) 上の `Run lint (Ruff)` ステップが依然として失敗している問題に対処します。

前回、`pyproject.toml` の設定を調整して一時的に既存のエラーを無視するようにしましたが、それに先立ってローカル環境で実行した `ruff check --fix` や `ruff format` によって一部最適化されたファイル群（例：未使用のimport `F401` の削除など、手動で無視設定に入れなかったエラーへの対策）が存在していました。
前回のコミットでは `pyproject.toml` のみをステージングしてしまったため、それらの自動修正済みファイルがリモートに送られておらず、CI環境では依然として未修正のソースコードに対してチェックが走り、失敗していることが判明しました。

## 提案される変更

### 1. 未コミットの自動修正差分のコミット
現在ローカルに残っている、以下の変更差分を含めてコミットします。
これらはすべて Ruff による安全な自動修正（未使用インポートの削除、空白調整など）とフォーマットです。
- [MODIFY] `backend/**/*.py` (変更が検知されているファイル群)

### 2. リモートへのプッシュ
- 本計画に沿って作成したブランチをリモートにプッシュし、Github Actions のパイプラインを走らせます。

## 検証計画
### 自動テスト
- ローカル環境での `uv run ruff check .` と `uv run pytest` はすでに全件緑色であることを確認済みです。
- リモートへプッシュ後、GitHub 上で Actions の結果が `Success` (Exit code: 0) になることを最終確認します。
