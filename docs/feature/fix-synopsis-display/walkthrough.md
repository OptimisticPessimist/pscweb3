# あらすじ表示不備の修正 修正確認 (Walkthrough)

あらすじ（シーン番号0）において、ト書き以外の要素（セリフや括弧書きなど）が出現しても、その後のト書きを継続して収集するように修正しました。

## 変更内容

### 1. バックエンド解析ロジックの修正 (`backend/src/services/fountain_parser.py`)
解析エンジン（`fountain_parser.py`）において、シーンの概要（`description`）を収集する際の終了条件を、あらすじシーン（シーン番号0）に限り緩和しました。

**修正の詳細:**
- 通常のシーンでは、ト書き以外の要素（登場人物、セリフ、括弧書き等）が現れた時点で `description` の収集を終了します（冒頭のト書きのみが概要となります）。
- **あらすじシーン（シーン0）**では、これらが出現しても収集フラグ（`collecting_description`）を維持し、シーン内の全てのト書きを `description` に蓄積し続けるようにしました。
- これにより、「ト書き -> セリフ -> ト書き」のような構造を持つあらすじでも、全てのト書きがデータベースに保存されるようになります。

## 検証プラン

### 手動確認
1. 修正後のコードをデプロイします。
2. あらすじにセリフや長いト書きを含むFountainファイルを**再アップロード**します。
3. 脚本ビューアーの「あらすじ」セクションに、1行目だけでなく全てのト書き内容が表示されることを確認します。

## 注意事項
この修正は**新規にアップロードまたは再アップロードされた脚本**に対して有効になります。既に解析済みのデータについては、再アップロードを行うことで反映されます。
