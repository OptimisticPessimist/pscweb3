# 実装計画 - あらすじ表示の修正

あらすじ（シーン#0）に含まれるキャラクター名、セリフ、ト書き、括弧書きなどの全要素が正確に抽出され、シーンの説明文（description）に統合されるように修正します。

## 1. 現状の課題
- `preprocess_fountain` があらすじセクション内の全行に `=` マーカーを強制付与していたため、キャラクターやセリフとして認識されず、単純なテキスト（Synopsis要素）として処理されていた。
- `fountain_parser.py` のあらすじ収集ロジックが断片的で、要素タイプによっては収集から漏れたり、フォーマットが不適切だった。
- キャラクター名とセリフが重複して説明文に追加されるケースがあった。

## 2. 修正方針
1. **`backend/src/utils/fountain_utils.py` の修正**
   - あらすじセクション内での `=` マーカーの強制付与を廃止し、通常の要素認識（Character, Dialogue, Action等）が行われるようにする。
   - ただし、明示的な `=` マーカーがある場合は引き続きサポートする。

2. **`backend/src/services/fountain_parser.py` の修正**
   - シーン#0（あらすじ）の場合、要素タイプに関わらず全てのテキストを `current_scene.description` に集約する。
   - セリフ要素については「名前: セリフ内容」の形式で整形して追加する。
   - キャラクター名単体の追加は、セリフとの重複を避けるためシーン#0ではスキップする。
   - 不要なデバッグ用 `print` 文を削除する。

## 3. 確認方法
- `repro_synopsis.py` を実行し、あらすじセクション内のセリフが「名前: セリフ」形式で含まれていること、および全要素が漏れなく説明文に含まれていることを確認する。
- 既存のユニットテスト（`test_fountain_parser.py` 等）を実行し、デグレがないことを確認する。
