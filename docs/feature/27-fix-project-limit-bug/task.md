# タスク: 公開脚本インポート時のプロジェクト制限バグ修正

## バグの詳細
`import_script`（公開脚本からのインポート）を使用してプロジェクトを作成する場合、プロジェクトが「公開」として作成されるため、プロジェクト作成数制限（非公開プロジェクトのみに適用）を回避できてしまいます。一方で、通常の `create_project` では、公開脚本から作成する場合は帰属保護のために強制的に「非公開」に設定されます。この不整合により、ユーザーは制限と帰属保護ポリシーを回避できてしまいます。
