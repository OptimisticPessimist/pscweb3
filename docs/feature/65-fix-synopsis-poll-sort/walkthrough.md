# 修正内容の確認 (Walkthrough)

## 修正の概要
あらすじ（Synopsis）の説明収集ロジックを修正して不要な文字情報が入り込まないようにし、香盤表のシーン表示順を Act および Scene の複合ソートに変更しました。

## 変更点
### バックエンド
- **`src/services/fountain_parser.py`**:
  - セクション見出し（`Section Heading`）に到達した際、および幕（Act）が切り替わった際に、シーン説明の収集を明示的に停止するようにしました。これにより、あらすじセクションの後に登場人物定義が続く場合に、その定義内容があらすじテキストとして扱われるのを防ぎます。
- **`src/api/scene_charts.py`**:
  - 香盤表の生成・取得時のソートロジックを修正しました。
- **`src/db/models.py`**:
  - 脚本からシーン、シーンからセリフのリレーションにあらかじめ `order_by` を設定し、DBからの取得時点で正しい順序が保証されるようにしました。
- **`src/schemas/scene_chart.py`**:
  - フロントエンドでの一意識別のために `scene_id` を追加しました。

### フロントエンド
- **`src/types/index.ts`**:
  - スキーマの変更を反映。
- **`src/features/scene_charts/pages/SceneChartPage.tsx`**:
  - 重複キー警告（複数の幕で同じシーン番号がある場合）を避けるため、`key` を `scene_id` に変更しました。

## 検証結果
- `pytest` により、複数のActがある場合や、あらすじの直後に登場人物セクションがある場合でも、正しくパースされ収集が停止することが確認されました。
