# タスク: RLSポリシーの最適化（過剰な権限の抑制）

## 概要
Supabase のセキュリティリンターより、`USING (true)` および `WITH CHECK (true)` を使用した `ALL` (全操作) 許可のポリシーがセキュリティリスクとして警告されている。
現在のバックエンドアーキテクチャでは、データの操作は主に FastAPI バックエンド（管理者権限でDB接続）が行うため、Supabase の公開 API (PostgREST) 経由での書き込み権限を制限し、警告を解消する。

## 詳細
- 全ての公開テーブルに対して設定されている "Enable all access" ポリシー（`FOR ALL`）を削除する。
- 代わりに `FOR SELECT` のみのポリシー（`USING (true)`）を設定し、読み取りは許可しつつ、API経由の書き込み・更新・削除を禁止する。
- `alembic_version` などのシステム管理テーブルについては、API経由のアクセス自体を不要であれば制限する。

## 完了条件
- [x] 全ての対象テーブルから `FOR ALL` の "Enable all access" ポリシーが削除されている。
- [x] 必要に応じて `FOR SELECT` のみのポリシーが設定されている。
- [x] Supabase のリンター警告 `rls_policy_always_true` が解消される（本番反映後）。
- [x] バックエンドアプリケーションの動作に影響がないことを確認する（バックエンドは通常RLSをバイパスするため問題ないはず）。
