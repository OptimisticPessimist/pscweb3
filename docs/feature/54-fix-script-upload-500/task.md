
# タスク: 台本アップロード時の500エラー修正

## 背景
ユーザーから台本（.fountain）のアップロード時に500エラーが発生するという報告があった。
調査の結果、以下の問題が判明した：
1. フロントエンドとバックエンドでファイルアップロード時のフィールド名が不一致（`script_file` vs `file`）。これにより本来は422エラーになるはずだが、環境によって500になる可能性がある。
2. バックエンドでのデコード処理（UTF-8固定）が例外を投げた際、現在の `try-except` ブロックの外側にあるためハンドルされず、500エラーとなる。日本語ユーザーはShift_JISやUTF-8 (BOMあり) を使用することが多いため、デコード失敗の可能性が高い。
3. `fountain` ライブラリの内部で、特定の入力（空行が含まれるヘッダーなど）に対して `IndexError` が発生する可能性がある。

## 作業内容
- [ ] バックエンドのデコード処理を改善し、例外を適切に捕捉して400エラー（不正なエンコーディング）として返すように修正。
- [ ] Shift_JISやUTF-8 (BOMあり) の自動判別・フォールバックを実装（`charset-normalizer` または `utf-8-sig`）。
- [ ] フロントエンドとバックエンドのフィールド名を `script_file` に統一、またはバックエンドで `file` 以外のエイリアスを許容。
- [ ] 修正内容の検証（テストコードの実行）。
- [ ] コードレビューの実施。
