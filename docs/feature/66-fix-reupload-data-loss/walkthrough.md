# 修正内容の確認: 脚本再アップロード後の日程調整シーン表示不具合の修正

脚本を再アップロードした際に、日程調整のカレンダー表示や候補日時詳細でシーンが表示されなくなる、あるいは順序が不自然になる不具合を修正しました。また、追加の要件に基づき、分析ロジックをより実用的な形に洗練させました。

## 修正内容

### Backend

#### [SchedulePollService](file:///f:/src/PythonProject/pscweb3-1/backend/src/services/schedule_poll_service.py)
- **あらすじの除外**: シーン番号が 0 以下のシーン（あらすじ等）を分析結果（稽古可能シーン、リーチ等）から完全に除外しました。
- **可用性判定の厳格化**: シーンに登場する全てのキャラクターに対して、少なくとも1人の配役済みキャストが参加可能（OK/Maybe）である場合のみ「稽古可能」と判定されるようにしました。
- **未配役キャラクターの扱い**: キャストが一人も割り当てられていないキャラクターがいるシーンは「不足」として扱い、稽古可能リストからは除外されます。
- **ダブルキャスト対応**: 1つの役に複数のユーザーが配役されている場合、誰か1人が参加できればシーンを可能として扱います。
- **表示形式の統一**: シーンの表示名称を `{Act}-{Scene}: {Heading}` 形式（例: `1-1: 稽古場`）に統一しました。
- **ソート順の修正**: 全てのシーン一覧および分析結果において、`act_number`（幕）と `scene_number` の順で正しくソートされるようにしました。

### 手動検証のポイント
- **おすすめ日程の表示**: `{Act}-{Scene}: {Heading}` 形式でシーンが表示されていることを確認してください。
- **あらすじの非表示**: カレンダーの「稽古可能シーン」の一覧からあらすじが消えていることを確認してください。
- **厳密な判定**: まだキャストが決まっていない役があるシーンや、キャストが欠席のシーンが「稽古可能」に含まれていないことを確認してください。
