# 修正内容の確認: 脚本再アップロード後の日程調整シーン見直し

脚本を再アップロードした際の日程調整の不具合修正と、分析ロジックをより実用的な形に洗練させました。

## 修正内容

### Backend

#### [SchedulePollService](file:///f:/src/PythonProject/pscweb3-1/backend/src/services/schedule_poll_service.py)
- **あらすじの除外**: シーン番号 0 以下のシーン（あらすじ等）を分析結果（稽古可能、もう一息）から完全に除外しました。
- **分析ロジックの再調整**:
    - **稽古可能（Possible）**: 配役済みの役者が全員揃っていれば、未配役のキャラクターがいても「稽古可能」として表示するようにしました。ただし、未配役がある場合は「未配役あり」という注釈を表示します。これにより、配役作業中でも稽古プランが立てやすくなります。
    - **もう一息（Ready to reach）**: 不足している役が**ちょうど1名**の場合のみを「もう一息」と判定するように厳格化しました。2名以上いない場合は調整の参考になりにくいため、リストをスッキリさせました。
    - **ダブルキャスト対応**: 1つの役に複数人割り当てられている場合、誰か1人が参加可能であればOKと判定します。
- **表示形式の統一**: シーンの表示名称を `{Act}-{Scene}: {Heading}` 形式に統一し、Act（幕）を考慮したソート順を保証しました。
- **UUIDの文字列変換**: APIレスポンスでの型不一致を防ぐための修正を行いました。

#### [SceneChartGenerator](file:///f:/src/PythonProject/pscweb3-1/backend/src/services/scene_chart_generator.py)
- **デバッグ情報の追加**: 進行状況を出力するデバッグプリントを追加し、処理の追跡を容易にしました。
