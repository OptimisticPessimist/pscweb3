# プロジェクト作成数制限機能 - Walkthrough

## 変更内容
ユーザーが作成（所有）できるプロジェクト数を最大2つまでに制限しました。

### Backend
- `src/api/projects.py`: `create_project` エンドポイントに、現在のユーザーが所有するプロジェクト数をチェックするロジックを追加。
  - 既に2つ以上のプロジェクトを所有している場合、HTTP 400エラー ("You can only own up to 2 projects.") を返します。

### Frontend
- `src/locales/*/translation.json`: 各言語に制限メッセージ (`dashboard.projectLimit` -> "作成できるプロジェクトは2つまでです" 等) を追加。
- `src/features/dashboard/DashboardPage.tsx`:
  - ユーザーが所有するプロジェクト数をカウント。
  - カウントが2以上の場合、"New Project" ボタンを無効化（グレーアウト）。
  - ボタンのツールチップおよび、プロジェクトがない状態の画面ではエラーメッセージとして制限理由を表示。

## 検証方法

### 1. 制限の確認
1. プロジェクトをまだ作成していない、または1つだけ所有しているユーザーでログイン。
2. ダッシュボードで「新規プロジェクト」ボタンが有効であることを確認。
3. プロジェクトを作成（2つ目）。
4. ダッシュボードに戻ると、「新規プロジェクト」ボタンが無効化（グレーアウト）され、クリックできないことを確認。
5. ボタンにカーソルを合わせると、「作成できるプロジェクトは2つまでです」等のメッセージが表示されることを確認。

### 2. オーナー以外のプロジェクト参加
1. 既に2つのプロジェクトを所有している状態で、別のユーザーからプロジェクトに招待される。
2. 招待を受け入れる。
3. プロジェクト一覧に3つ以上のプロジェクトが表示されるが、「新規プロジェクト」ボタンは無効のままであることを確認（所有プロジェクト数のみが制限対象であるため）。

### 3. プロジェクト削除
1. 所有するプロジェクトを1つ削除する。
2. 「新規プロジェクト」ボタンが再び有効になることを確認。
