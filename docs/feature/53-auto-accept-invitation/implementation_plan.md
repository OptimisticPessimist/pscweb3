# 実装計画：招待リンクからの自動プロジェクト参加

ユーザーが「プロジェクトに招待されています」の画面からDiscordログインを行った後、再度「プロジェクトに招待されています」という画面をスキップして、一気にプロジェクト参加処理へ進むための実装計画です。

## User Review Required
既存の「ログイン完了後に元の画面へ戻す」という汎用的なリダイレクト機能（`postLoginRedirect`）を利用していましたが、招待に特化した処理がありませんでした。
今回は、URLにクエリパラメータ（`?auto_accept=true`）を含めてリダイレクトさせることで、ログイン後に招待ページへ戻ってきた際に、自動的に参加処理APIを叩くように変更します。
この方針で実装してよろしいでしょうか？

## Proposed Changes

### フロントエンド
#### [MODIFY] `frontend/src/features/projects/pages/InvitationLandingPage.tsx`
- **変更内容**:
  - `handleLogin` 関数内で `postLoginRedirect` に保存するパスに `?auto_accept=true` を付与するよう変更。
  - カスタムフックや `useEffect` を用いて、初期レンダリング時かつ `user` が存在（ログイン済み）で、かつ URL クエリパラメータに `auto_accept=true` が存在する場合、自動的に `acceptMutation.mutate()` を実行するロジックを追加。
  - （オプション）自動処理中であることがユーザーに伝わるように、`isPending` のときに「参加処理中...」といったローディングUIを表示する。

## Verification Plan
1. **マニュアルテスト**:
   - シークレットタブ等で非ログイン状態で立ち上げ、招待URL（`/invitations/...`）へのアクセスを行う。
   - 画面の「Discordでログイン」ボタンを押下してログインフローを完了する。
   - リダイレクト後、再度「プロジェクトに招待されています」画面で手動でボタンを押すことなく、一気にプロジェクト詳細画面（`/projects/...`）へ遷移することを確認する。
