# 修正内容の確認 (Walkthrough)

招待リンクからのプロジェクト参加フローを改善し、ログイン後の二度手間を解消しました。

## 変更内容
### フロントエンド
#### [InvitationLandingPage.tsx](file:///f:/src/PythonProject/pscweb3-1/frontend/src/features/projects/pages/InvitationLandingPage.tsx)
- **Discordログイン遷移の変更**:
  - `localStorage` に保存するリダイレクト先URLに `?auto_accept=true` を付与するようにしました。
- **自動参加ロジックの追加**:
  - `useEffect` を使用して、ログイン済みかつ `auto_accept=true` パラメータが存在する場合に、自動的に招待承諾API (`acceptMutation.mutate()`) を呼び出すようにしました。
- **UIの改善**:
  - 自動参加処理中（API呼び出し中）は、ボタンの代わりに「参加処理中...」というスピナー付きのメッセージを表示するようにし、ユーザーに状況が伝わるようにしました。

## 検証結果
### 手動検証
以下の手順で動作を確認しました。
1. 非ログイン状態で招待URLにアクセス。
2. 「Discordでログイン」ボタンをクリック。
3. Discord認証完了後、招待ページにリダイレクトされる。
4. **（自動処理）** 画面が表示された直後に自動的に参加処理が開始され、スピナーが表示される。
5. **（自動処理）** 参加成功後、即座にプロジェクト詳細画面へ遷移する。

これにより、ユーザーが手動で再度「プロジェクトに参加」ボタンを押す必要がなくなり、UXが向上しました。
