# 実装計画: あらすじ機能追加後のテスト失敗修正

## 1. 現状の課題
あらすじ（Synopsis）情報の抽出ロジックを改善した際、以下の問題が発生し、既存のテストが失敗するようになった。
- **`preprocess_fountain` の副作用**: 標準的なシーン見出し（`INT.` など）がアクションとして誤認され、`!` が付与されることでパースが壊れていた。
- **`fountain-python` の制限**: 連続した強制アクション行（`!` で始まる行）の間に空行がないと、2行目以降が消失する。これにより登場人物定義が欠落していた。
- **重複したあらすじ作成**: テキストの位置によっては、あらすじ（Scene 0）が複数作成され、DBの制約やテストの期待値に反していた。
- **状態管理の不備**: 登場人物セクションを抜けた後もフラグが残り続け、後続のテキストが誤って処理されていた。
- **幕番号の欠落**: 幕（Act）を指定した直後のシーンで幕番号が `None` になるケースがあった。

## 2. 修正方針
- **前処理 (`preprocess_fountain`) の強化**: 
    - シーン見出しの検出精度を上げ、誤った `!` 付与を防ぐ。
    - 登場人物セクション内の各行の間に強制的に空行を入れ、データの消失を防ぐ。
    - シーン見出しを検出した際に、セクション状態を確実にリセットする。
- **パーサー (`fountain_parser.py`) の改善**:
    - あらすじ（Synopsis）が既に作成されている場合、自動作成ロジックをスキップして重複を防ぐ。
    - パースのメインループで、前もって処理済みの登場人物セクションを明示的にスキップする。
    - 幕番号のカウンターを、あらすじ処理などの間も保持するように修正。

## 3. 修正詳細
### `backend/src/utils/fountain_utils.py`
- `is_in_character_section` フラグを導入。
- `is_all_caps` に `:` のチェックを追加し、定義行（NAME: Desc）との混同を避ける。
- `@Name Dialogue` の一行書きにおいて、名前の一部としてのカッコ（例：`@花子 (ハナコ)`）を dialogue と誤認しないように正規表現を調整。

### `backend/src/services/fountain_parser.py`
- `synopsis_scene_already_exists` フラグを導入。
- 3つ目のループ（本文抽出）で `in_ignored_section` を管理し、登場人物セクションをスキップ。
- セクションシーン（`##`）とスラッグライン（`INT.`）の結合ロジックにおいて、間にト書き（Action）があっても結合を維持するように微調整（テストの期待値に合わせる）。

## 4. 期待される結果
- 失敗していた 5 つのユニットテストがすべて合格する。
- 既存の全ユニットテスト (122件) が合格し、デグレが発生していないことを確認する。
