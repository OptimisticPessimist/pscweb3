# Azure Functions 404エラーおよび500エラー調査・修正

## 概要
Azure Functions上で `/auth/login` が404エラー、およびデバッグ等の過程で500エラーになる問題を解決しました。

## 原因
1. **根本原因**: `backend/src/api/rehearsals.py` に `SyntaxError`（閉じられていない括弧）が存在し、FastAPIアプリケーションの起動（インポート）に失敗していました。
2. **404エラー**: アプリ起動失敗により、Azure FunctionsがHTTPトリガーを見つけられず、404を返していました。

## 対処内容

1. **Backend (`function_app.py`) の堅牢化**:
   - アプリケーション起動時のインポートエラーをキャッチし、500エラーとして詳細なトレースバックを返すように修正しました。これにより、Azureのログを見なくてもブラウザ上で「何が原因で起動していないか」を確認できるようになりました。

2. **Backend (`rehearsals.py`) の修正**:
   - `SyntaxError` を修正し、正常にインポートできるようにしました。

3. **クリーンアップ**:
   - 調査用に追加した `/debug` エンドポイントは削除しました（エラーハンドリング機能は維持）。

## 検証結果
- `/auth/login`: 正常にDiscord認証画面へ遷移することを確認。
